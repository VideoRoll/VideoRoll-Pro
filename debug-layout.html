VideoRoll-Pro Layout Configuration Debug

问题分析

现在的情况是：

  • ✅ Popup可以正常显示
  • ❌ 保存配置后popup布局没有变化

可能的原因

  • 配置加载时机问题： Popup可能在配置保存之前就加载了旧配置
  • 缓存问题： 单例模式可能导致配置被缓存，没有重新读取storage
  • 响应式更新问题： 配置更新后计算属性没有重新计算
  • Storage同步问题： 保存和读取可能存在时序问题

解决方案

已实施的修复：

  1. 单例模式： 确保配置在全局共享
  2. 强制重新加载： Popup每次打开都直接从storage读取最新配置
  3. 响应式更新： 使用reactive对象确保UI自动更新
  4. 调试信息： 添加console.log跟踪配置加载过程

测试步骤

  1. 打开扩展的Options页面
  2. 进入"布局配置"选项卡
  3. 修改一些组件的可见性或布局
  4. 点击"保存"按钮
  5. 打开popup，检查布局是否发生变化
  6. 查看浏览器控制台的调试信息

关键修改文件

  • useLayoutConfig.ts: 改为单例模式，确保配置全局共享
  • useLayoutComponents.tsx: 每次popup打开强制重新加载配置
  • GridPanel/index.tsx: 使用新的布局组件系统

调试信息检查

在浏览器控制台中查找以下信息：

---Force reloaded config from storage: [配置对象]
---Computing components, isConfigLoaded: true, tabs length: [数量]
Layout config loaded: [配置对象]

如果看到这些信息，说明配置加载正常。如果没有看到，说明还有其他问题。

预期结果

完成上述修改后，应该可以实现：

  • 在Options页面修改布局配置
  • 点击保存按钮
  • 打开popup时看到更新后的布局
  • 配置在浏览器重启后依然保持
